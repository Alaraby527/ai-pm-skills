# 数据分析 Skill

## 简介

**干什么的**：产品数据分析方法论与三大分析模型组合工作流。从指标监控到根因定位，覆盖事件分析、漏斗分析、分层拆解三大核心模型，以及 RFM 用户价值分层、数据异常诊断、指标体系搭建等方法。

**核心方法论**：三大模型组合策略（事件监控发现异常 → 漏斗看转化流失 → 分层拆解定位根因）+ RFM 用户价值分层法 + 数据异常诊断框架 + 指标体系搭建方法。

**你能得到什么**：数据异常根因分析、漏斗分析报告、用户分层结果、指标体系设计、数据周报/月报、A/B 测试分析结论。

## 触发场景

当用户需要分析产品数据、定位数据异常、输出分析报告时触发。典型场景包括：
- 某个指标下跌了，想知道为什么
- 搭建产品数据指标体系
- 做漏斗分析、用户分层分析
- 写数据周报/月报/分析报告
- A/B 测试结果分析

## 输入要求

**必需信息**：
- 业务问题或分析目标（要分析什么）
- 数据（数据表格 / 数据描述 / 上传数据文件）

**可选补充信息**：
- 产品类型和业务模式
- 已知的背景信息（最近做了什么变更）
- 时间范围

如果信息不全，通过对话逐步澄清。

## 工作流（5 步）

### Step 1：明确分析目标与问题定义

- 明确要回答的业务问题（1-3 个核心问题）
- 判断问题类型：
  - **监控型**：指标涨了/跌了多少？
  - **诊断型**：为什么涨/跌？根因是什么？
  - **决策型**：应该怎么做？有什么建议？
- 确定分析的时间范围和颗粒度
- 确认数据是否可得

### Step 2：事件分析——看大盘，发现异常

**事件分析基础**：
- 核心指标是什么（北极星指标 + 一级指标）
- 指标怎么算（定义、口径、分子分母）
- 看趋势：日/周/月趋势、同比、环比

**异常识别**：
- 指标是否在正常波动范围内？
- 和前一周期相比变化了多少？
- 是突增突降还是持续趋势？
- 从什么时候开始变化的？（拐点定位）

**常用指标类型**：
| 指标类别 | 示例 |
|---------|------|
| 用户规模 | DAU/MAU、新增用户、留存率 |
| 活跃/参与 | 人均使用时长、功能使用率、访问深度 |
| 转化 | 注册转化率、付费转化率、功能渗透率 |
| 商业化 | 收入、ARPU、付费率、LTV |
| 内容/供给 | 内容生产量、消费量、互动率 |

### Step 3：漏斗分析——看转化，找到流失点

**漏斗搭建方法**：
1. 确定核心用户路径（用户从进入到完成目标的完整流程）
2. 拆分关键节点（每个有数据的环节）
3. 计算各层转化率（每层人数 / 上一层人数）
4. 找到流失最大的环节（掉得最多的那一步）
5. 进一步分析流失原因

**常见漏斗类型**：
- 新用户注册漏斗（访问 → 注册 → 激活 → 留存）
- 付费转化漏斗（浏览 → 加购 → 下单 → 支付 → 复购）
- 功能使用漏斗（进入 → 点击 → 完成 → 复用）

**漏斗分析注意事项**：
- 时间窗口定义要清晰（同一用户 7 天内 / 单次会话内）
- 分子分母要对齐（同周期进入的用户走完整流程）
- 不仅看整体转化率，还要看每层流失率

### Step 4：分层拆解——下钻定位根因

> 事件分析发现异常 → 漏斗分析找到流失环节 → 分层拆解定位是哪群人/哪个渠道的问题

**维度一：用户维度分层**

| 分层方式 | 说明 | 适用场景 |
|---------|------|---------|
| **渠道分层** | 按获客渠道拆分（自然/投放/邀请等） | 新增/留存变化时 |
| **地域分层** | 按城市级别/地区拆分 | 本地化相关产品 |
| **行为分层** | 按是否使用核心功能、是否付费拆分 | 功能渗透率、付费分析 |
| **RFM 分层** | 按消费行为三维度打标签 | 电商/付费产品用户价值分析 |
| **版本分层** | 按产品版本拆分 | 版本迭代效果评估 |
| **新鲜度分层** | 按注册时长拆分 | 新老用户差异分析 |

**维度二：供给维度分层**（电商/内容产品适用）
- 按品类拆分（3C / 服饰 / 食品…）
- 按内容垂类拆分（影视 / 知识 / 娱乐…）
- 按价格带拆分

**分层拆解方法**：
1. 先假设一个影响维度（比如"是不是渠道质量下降了？"）
2. 按那个维度拆分指标
3. 看各细分维度的变化幅度
4. 找到变化最大的那个细分维度 → 可能就是根因
5. 如果一个维度不够，换另一个维度，或者多维度交叉

**RFM 用户价值分层四步流程**：
1. 提取历史订单数据，整理 R/F/M 原始数据
2. 按分位数给三维度打分（1-5 分）
3. 计算三维度均值，区分「高/低」两档
4. 2×2×2 组合出 8 类价值用户（重要价值、重要发展、重要保持、重要挽留、一般价值、一般发展、一般保持、一般挽留）

### Step 5：输出分析报告

按 `references/template-analysis-report.md` 模板输出，包含：
- 分析背景与目标
- 核心结论（先给结论，再展开）
- 详细分析（事件 + 漏斗 + 分层）
- 原因分析与验证
- 行动建议
- 后续跟进指标

## 三大模型组合使用策略

**产品数据完整分析链路**：
> 事件监控发现异常 → 漏斗看转化流失 → 分层拆解定位根因

| 问题 | 先用什么模型 | 再用什么模型 | 最后用什么模型 |
|------|-------------|-------------|---------------|
| DAU 跌了，为什么？ | 事件分析（看趋势、看拐点） | 分层拆解（按渠道/版本/新老用户） | 漏斗分析（看是哪个环节掉了） |
| 转化率下降 | 漏斗分析（找到流失最大的环节） | 分层拆解（看哪类用户掉得多） | 事件分析（看该环节的具体指标） |
| 收入下降 | 事件分析（看整体趋势） | 分层拆解（按用户价值/品类） | 漏斗分析（付费漏斗哪层掉了） |
| 留存率下跌 | 事件分析（看各日留存曲线） | 分层拆解（按渠道/版本/行为） | 漏斗分析（新手流程哪步流失） |
