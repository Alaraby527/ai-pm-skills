# 数据分析提示词合集

## 1. 指标体系搭建

### 1.1 北极星指标设计提示词

```
我正在做一款【产品类型，如"AI 写作工具"】，请帮我设计一套数据指标体系。

产品描述：
【产品做什么、目标用户、商业模式】

请输出：
1. 北极星指标（一个最核心的指标）及理由
2. 一级指标（3-5 个，支撑北极星指标）
3. 二级指标（每个一级指标下的拆解指标）
4. 指标树（从上到下的关联关系）
5. 不同阶段（MVP/成长期/成熟期）关注的重点指标变化

要求：
- 指标要可测量、有业务含义
- 遵循 AARRR 或类似的框架，但不要生搬硬套
- 结合 AI 产品的特点（如果有 AI 相关的特殊指标）
```

### 1.2 指标体系健康度诊断提示词

```
我们产品的核心指标有：
【列出当前在看的指标】

产品类型是：【产品类型】
当前阶段是：【MVP/成长期/成熟期】

请帮我诊断这套指标体系：
1. 缺了什么重要指标？
2. 哪些指标是重复的/冗余的？
3. 指标之间的因果关系清晰吗？
4. 有没有"看起来好看但没什么用"的虚荣指标？
5. 如果只保留 5 个最核心的指标，你会留哪 5 个？为什么？
```

## 2. 异常诊断

### 2.1 数据异常诊断思路提示词

```
我的产品【指标名称】突然跌了/涨了，具体情况是：
【描述异常情况：什么指标、变化了多少、从什么时候开始的】

已知背景信息：
【最近有没有上线新功能、有没有活动、有没有外部因素...】

请帮我梳理一个分析思路：
1. 首先应该确认什么？（数据是不是真的？口径有没有变？）
2. 然后从哪几个维度拆解？（按优先级排序）
3. 每个维度怎么判断是不是根因？
4. 需要验证哪些假设？
5. 如果前几个维度都解释不了，接下来怎么办？

输出格式：分析步骤清单，每步说明：做什么、看什么、怎么判断、如果是/不是下一步怎么办。
```

### 2.2 DAU 下跌分析框架提示词

```
【产品名称】的 DAU 最近下跌了 XX%，从【日期】开始的。

请帮我设计一个完整的分析框架，用于定位 DAU 下跌的原因：

1. **数据校验层**（先排除数据问题）
   - 看什么：
   - 怎么判断：

2. **用户结构拆解**（看是哪类用户掉了）
   - 维度 1：新用户 vs 老用户
   - 维度 2：渠道维度
   - 维度 3：版本维度
   - 维度 4：地域维度
   - 维度 5：用户活跃度分层
   - 每个维度：看什么指标、怎么判断是不是这个维度的问题

3. **行为链路拆解**（看用户在哪一步掉了）
   - 访问 → 打开 APP → 使用核心功能 → 次日留存
   - 哪个环节的转化率变化最大？

4. **供给/内容侧排查**（如果有供给的话）
   - 内容/商品供给有没有变化？
   - 质量有没有变化？

5. **外部因素排查**
   - 竞品动态？
   - 节假日/季节性？
   - 政策/舆情？

6. **验证与结论**
   - 怎么确认根因？
   - 如何排除其他因素？

请尽量具体，不要太笼统。
```

### 2.3 转化率下降分析提示词

```
【漏斗名称】的转化率从 XX% 降到了 XX%，下降了 XX pp。

漏斗的步骤是：
步骤 1 → 步骤 2 → 步骤 3 → ... → 最终转化

请帮我分析：
1. 首先应该看哪个步骤的变化最大？
2. 找到掉得最多的步骤后，怎么进一步分析原因？
3. 需要按哪些维度拆分来定位人群？
4. 常见的转化率下降原因有哪些？（列 10 个以上，并说明怎么验证）
5. 如果是产品改版后下降的，怎么确认是改版导致的？

输出格式：原因假设清单，每个假设包含：假设内容、验证方法、数据需求。
```

## 3. 漏斗分析

### 3.1 漏斗设计提示词

```
我想分析【场景，如"新用户从注册到首次使用核心功能"】的转化率。

产品/功能描述：
【描述产品和用户路径】

请帮我：
1. 设计这个漏斗的步骤（5-7 个步骤最合适）
2. 每个步骤的精确定义（什么事件/行为算完成了这一步）
3. 时间窗口建议（多长时间内完成算有效转化）
4. 这个漏斗的基准转化率大概是多少（行业参考）
5. 哪些步骤是关键流失点（通常流失最多的）

要求步骤定义清晰、可被数据埋点追踪。
```

### 3.2 漏斗优化建议提示词

```
这是我们产品【漏斗名称】的数据：

第 1 步：10,000 人（100%）
第 2 步：6,500 人（65%）
第 3 步：4,200 人（64.6%）
第 4 步：2,800 人（66.7%）
第 5 步：840 人（30%）
最终转化率：8.4%

请分析：
1. 哪个环节流失最严重？
2. 每个环节的流失可能原因是什么？
3. 优化优先级怎么排？（考虑效果和成本）
4. 具体的优化建议有哪些？（每个流失点至少 2-3 条）
5. 如果能把最大流失点的转化率提升 10 pp，整体转化率能提升多少？
```

## 4. 用户分层分析

### 4.1 留存分析思路提示词

```
我们产品的次日留存最近下降了，从 XX% 降到了 XX%。

请帮我设计一个留存分析框架：
1. 先看什么？（整体留存曲线长什么样？）
2. 怎么判断是新用户留存问题还是老用户留存问题？
3. 应该按哪些维度拆分来定位？（至少 5 个维度）
4. 如果是新用户留存掉了，常见原因有哪些？怎么排查？
5. 如果是老用户留存掉了，常见原因有哪些？怎么排查？
6. 怎么区分是"产品本身问题"还是"渠道质量问题"？

每个点要具体，说清楚看什么数据、怎么判断。
```

### 4.2 RFM 分析提示词

```
我想给我们的用户做 RFM 分层。产品类型是【电商/订阅制/SaaS 等】。

请帮我：
1. RFM 三个维度在我们这个业务里分别怎么定义最合适？
2. 时间周期选多长比较合理？
3. 打分用分位数还是固定阈值？各有什么优缺点？
4. 八类用户的运营策略分别是什么？（结合我们的业务特点）
5. 除了 RFM，还有哪些有价值的用户分层维度可以加？

【业务补充信息：】
```

## 5. A/B 测试分析

### 5.1 A/B 测试结果解读提示词

```
我们做了一个 A/B 测试，结果如下：

【实验名称：】
实验时间：
分组：A 组（对照组）XX 人，B 组（实验组）XX 人
核心指标：
  - A 组：XX
  - B 组：XX
  - 变化：+XX% / -XX%
  - p 值：XX（或置信区间：XX）
其他指标：
  - 指标 1：...
  - 指标 2：...

请帮我解读：
1. 这个结果显著吗？能不能说 B 组比 A 组好/差？
2. 除了核心指标，其他指标有没有异常？（有没有"按下葫芦浮起瓢"）
3. 这个结果可靠吗？需要注意什么问题？（比如样本量够不够、有没有辛普森悖论）
4. 要不要全量上线？为什么？
5. 如果要继续实验，建议怎么做？

如果数据不足请指出需要补充什么信息。
```

## 6. 报告输出

### 6.1 分析报告生成提示词

```
请根据以下分析结果，生成一份专业的数据分析报告：

【分析背景：】
【核心发现：】
【数据摘要：】
【原因分析：】
【建议：】

报告要求：
1. 先给结论，再展开分析（金字塔原则）
2. 结构清晰，分章节
3. 每个结论有数据支撑
4. 建议具体可执行，不说空话
5. 语言专业、精炼，不超过 2000 字

输出格式：
- 标题
- 核心结论（3 条，每条 1-2 句话）
- 详细分析（分 2-3 个小节）
- 原因与验证
- 行动建议（分 P0/P1/P2）
- 后续跟进
```
