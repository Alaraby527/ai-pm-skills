# RFM 用户价值分层模板

## RFM 模型说明

### 什么是 RFM
RFM 是基于用户消费行为的三维度用户价值分层模型：

| 维度 | 英文 | 含义 | 说明 |
|------|------|------|------|
| **R** | Recency | 最近一次消费间隔 | 数值越大 = 越久没消费 = 越危险 |
| **F** | Frequency | 周期内消费频次 | 数值越大 = 越活跃 = 价值越高 |
| **M** | Monetary | 周期内消费总金额 | 数值越大 = 消费能力越强 = 价值越高 |

### 适用场景
- 电商、订阅制、付费产品的用户价值分层
- 用户运营（不同用户不同运营策略）
- 营收分析（哪类用户贡献最大）
- 流失预警（R 值变大的用户）

## 一、数据准备

### 1.1 数据来源
- 订单表 / 支付记录表
- 用户表

| 字段 | 说明 |
|------|------|
| user_id | 用户 ID |
| last_order_date | 最近一次消费日期 |
| order_count | 周期内订单数 |
| total_amount | 周期内消费总金额 |

### 1.3 时间周期
- 分析周期：【如"近 90 天"】
- 数据截止日期：【日期】

## 二、打分规则

### 2.1 分位数设定
按分位数给每个维度打 1-5 分（5 分最高）。

**R 维度（最近消费间隔）**
> 注意：R 值越小越好（最近刚消费），所以打分是反向的

| 分位数 | R 值范围（天） | 得分 |
|--------|---------------|------|
| 0-20% | 0 - X 天 | 5 分 |
| 20-40% | X - Y 天 | 4 分 |
| 40-60% | Y - Z 天 | 3 分 |
| 60-80% | Z - W 天 | 2 分 |
| 80-100% | > W 天 | 1 分 |

**F 维度（消费频次）**

| 分位数 | F 值范围（次） | 得分 |
|--------|---------------|------|
| 0-20% | | 1 分 |
| 20-40% | | 2 分 |
| 40-60% | | 3 分 |
| 60-80% | | 4 分 |
| 80-100% | | 5 分 |

**M 维度（消费金额）**

| 分位数 | M 值范围（元） | 得分 |
|--------|---------------|------|
| 0-20% | | 1 分 |
| 20-40% | | 2 分 |
| 40-60% | | 3 分 |
| 60-80% | | 4 分 |
| 80-100% | | 5 分 |

> 分位数取值参考二八法则：20% 的用户贡献 80% 的营收。
> 实际操作中，根据数据分布调整分位线。

## 三、用户分层结果

### 3.1 高低档划分
计算每个用户的 R/F/M 得分后，与各维度平均分比较：
- 得分 ≥ 平均分 = 高
- 得分 < 平均分 = 低

### 3.2 八类用户组合

| 用户类型 | R | F | M | 特征 | 运营策略 |
|---------|---|---|---|------|---------|
| **重要价值用户** | 高 | 高 | 高 | 最近常买、频次高、花钱多 | VIP 服务、专属权益、新品优先体验 |
| **重要发展用户** | 高 | 低 | 高 | 最近买过、花钱多但频次低 | 提升频次：发复购券、推荐相关商品 |
| **重要保持用户** | 低 | 高 | 高 | 以前常买且花钱多，但最近没买了 | 召回：定向活动、专属优惠、唤醒邮件 |
| **重要挽留用户** | 低 | 低 | 高 | 以前花得多，但最近不买了、频次也降了 | 紧急挽留：电话回访、高额优惠、调研流失原因 |
| **一般价值用户** | 高 | 高 | 低 | 最近常买、频次高但客单价低 | 提升客单价：满减券、高价商品推荐 |
| **一般发展用户** | 高 | 低 | 低 | 最近买过，但频次和金额都低 | 培养：新人礼包、引导多次购买 |
| **一般保持用户** | 低 | 高 | 低 | 以前买得多，但最近没买、客单价低 | 普通召回：通用优惠券、活动推送 |
| **一般挽留用户** | 低 | 低 | 低 | 各方面都差，低价值用户 | 减少投入，随缘运营，甚至放弃 |

## 四、运营策略落地

### 4.1 按用户类型的运营动作

**重要价值用户（VIP）**
- 专属客服/客户经理
- 新品优先体验权
- 专属活动和优惠
- 生日/节日关怀
- 邀请加入高端会员群

**重要发展用户（高价值但低频）**
- 定期推荐个性化商品/服务
- 发放复购优惠券（有时间限制，刺激尽快消费）
- 会员等级升级激励
- 关联推荐（购买了 A，推荐 B）

**重要保持用户（有流失风险的高价值）**
- 定向召回活动（专属优惠力度大）
- 新品/新功能优先通知
- 邮件/短信唤醒
- 客服主动回访（了解原因）

**重要挽留用户（高危高价值）**
- 最高优先级挽留
- 专人电话回访（了解流失原因）
- 超大力度优惠
- 调研问卷（了解为什么不来了）
- 产品改进后定向通知

**一般价值用户（高频低客单）**
- 满减券、凑单推荐（提升客单价）
- 升级引导（引导消费更高价位商品）
- 组合套餐推荐

## 五、计算示例（Python 伪代码）

```python
import pandas as pd
import numpy as np

# 假设 df 是用户消费数据，包含 user_id, recency_days, frequency, monetary

# R 打分（反向：天数越少分越高）
df['r_score'] = pd.qcut(df['recency_days'], 5, labels=[5,4,3,2,1]).astype(int)

# F 打分
df['f_score'] = pd.qcut(df['frequency'].rank(method='first'), 5, labels=[1,2,3,4,5]).astype(int)

# M 打分
df['m_score'] = pd.qcut(df['monetary'].rank(method='first'), 5, labels=[1,2,3,4,5]).astype(int)

# 高低划分
r_avg = df['r_score'].mean()
f_avg = df['f_score'].mean()
m_avg = df['m_score'].mean()

df['r_high'] = df['r_score'] >= r_avg
df['f_high'] = df['f_score'] >= f_avg
df['m_high'] = df['m_score'] >= m_avg

# 分类
def classify(row):
    r, f, m = row['r_high'], row['f_high'], row['m_high']
    if r and f and m: return '重要价值用户'
    elif r and not f and m: return '重要发展用户'
    elif not r and f and m: return '重要保持用户'
    elif not r and not f and m: return '重要挽留用户'
    elif r and f and not m: return '一般价值用户'
    elif r and not f and not m: return '一般发展用户'
    elif not r and f and not m: return '一般保持用户'
    else: return '一般挽留用户'

df['segment'] = df.apply(classify, axis=1)
```
