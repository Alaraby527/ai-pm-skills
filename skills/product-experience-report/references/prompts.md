# 产品体验分析提示词合集

## 1. 整体分析

### 1.1 五要素全面分析提示词

```
请帮我用用户体验五要素模型分析【产品名称】这款产品。

我已经体验了一段时间，以下是我的观察和感受：
【粘贴你的体验笔记/截图描述/功能清单】

请从五个层次逐一分析：
1. 战略层：产品定位、目标用户、核心需求、商业模式
2. 范围层：功能清单、核心功能、功能取舍、AI能力（如果有）
3. 结构层：模块划分、导航设计、核心用户流程
4. 框架层：首页布局、信息层级、交互设计
5. 表现层：视觉风格、配色、品牌、细节

每个层次：
- 先描述事实（产品怎么做的）
- 再评价好坏（为什么好/为什么不好）
- 最后给改进建议

最后给一个综合评分和总结。
```

### 1.2 快速体验报告提示词

```
请帮我快速分析【产品名称】，输出一份精简的体验报告。

要求：
1. 500 字以内
2. 包含：定位、核心功能、最大亮点、最大问题、整体评价
3. 语言精炼，观点明确

我的初步印象：【可选，填你自己的感受】
```

## 2. 分层分析提示词

### 2.1 战略层分析提示词

```
【产品名称】，请帮我做战略层分析：

1. 这个产品的定位是什么？（一句话描述）
2. 它的目标用户是谁？有什么特征？
3. 它解决了用户的什么核心需求？是刚需还是锦上添花？
4. 它的商业模式是什么？公司为什么做这个产品？
5. 和同类产品相比，它的差异化是什么？
6. 战略层面有什么问题或隐患吗？

产品背景：【补充信息】
同类产品有：【可选】
```

### 2.2 范围层分析提示词

```
【产品名称】的功能清单如下：
【列出功能清单】

请帮我做范围层分析：
1. 这些功能可以分成几个模块？每个模块的核心价值是什么？
2. 哪些是核心功能（用户为了它而用）？哪些是次要功能？
3. 功能和产品定位匹配吗？有没有"多余"的功能？
4. 有没有该做但没做的功能？为什么没做？
5. 功能取舍体现了什么产品思路？
6. AI 功能（如果有）做得怎么样？是真有用还是噱头？
```

### 2.3 结构层分析提示词

```
我正在分析【产品名称】的结构层。产品的模块和导航如下：
【描述模块划分和导航方式】

核心任务【任务名称】的流程是：
【描述流程步骤】

请帮我分析：
1. 模块划分合理吗？职责清晰吗？有没有重叠？
2. 导航设计怎么样？用户找功能容易吗？
3. 核心任务流程顺畅吗？能不能简化？
4. 信息架构怎么样？分类逻辑用户容易理解吗？
5. 如果是你重新设计结构，你会怎么改？为什么？
```

### 2.4 框架层分析提示词

```
这是【产品名称】的首页布局描述：
【从上到下详细描述首页有什么】

请帮我做框架层分析：
1. 信息层级清晰吗？用户一眼知道看哪里吗？
2. 重要功能/信息放在合适的位置了吗？
3. 每个区域的设计目的是什么？设计得好不好？
4. 有没有信息过载或重点不突出的问题？
5. 搜索入口在哪？设计得合理吗？
6. 整体来看，首页布局最大的优点和最大的问题分别是什么？
```

### 2.5 表现层分析提示词

```
【产品名称】的视觉设计：
【描述你看到的——配色、风格、字体、动效、细节等】

请帮我做表现层分析：
1. 整体视觉风格是什么？用 3 个关键词形容
2. 这种风格和产品定位、目标用户匹配吗？为什么？
3. 配色方案怎么样？舒服吗？有品牌识别度吗？
4. 动效用得怎么样？是加分还是干扰？
5. 细节品质如何？（圆角、阴影、间距、图标等）
6. 如果让你给视觉设计打分（1-10），你打几分？为什么？
```

## 3. 对比分析

### 3.1 两款产品横向对比提示词

```
请用用户体验五要素对比分析【产品 A】和【产品 B】：

【产品 A 的信息：】
【产品 B 的信息：】

对比维度：
1. 战略层：定位、用户、需求、商业模式
2. 范围层：功能覆盖、核心功能、功能取舍
3. 结构层：模块划分、导航、核心流程
4. 框架层：布局、交互、信息层级
5. 表现层：视觉、配色、品牌

每个维度给出：
- 对比表格
- 各自的优势
- 各自的劣势
- 你更看好哪个？为什么？
```

## 4. 面试答题版

### 4.1 面试题：分析一款你常用的产品

```
请帮我准备"分析一款你常用的产品"的面试回答。

产品名称：【产品名】
面试岗位：AI 产品经理

要求：
1. 总时长 3-5 分钟的回答量（大约 800-1200 字）
2. 结构清晰，用五要素但不要说"五要素"这三个字
3. 有观点有洞察，不只是描述
4. 最后要有对产品的建议（体现产品思维）
5. 语言口语化，适合面试说

重点突出 AI 相关的分析，因为面的是 AI PM。
```

### 4.2 面试题：如果你来做我们的产品

```
我面试的岗位是【岗位名称】，产品是【产品类型】。

面试题："如果让你来做我们这款产品，你会怎么做？"

请帮我准备答题框架和思路：

要求：
1. 不要上来就说功能，要先讲思路
2. 结构：先定位问题 → 分析原因 → 给出方案 → 说明优先级
3. 体现产品思维（用户价值、商业价值、可行性平衡）
4. 有数据意识（怎么验证）
5. 2-3 分钟的回答量

注意：不要凭空捏造具体功能，而是给出思考框架，显得有方法论。
```
