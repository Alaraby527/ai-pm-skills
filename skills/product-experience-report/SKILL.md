# 产品体验报告 Skill

## 简介

**干什么的**：基于用户体验五要素的产品体验分析方法论工具包。从战略层到表现层五层拆解，系统性分析一款产品的设计思路和体验优劣。

**核心方法论**：用户体验五要素模型（战略层→范围层→结构层→框架层→表现层）+ 五要素逐层分析 checklist + 综合总结框架。

**你能得到什么**：完整产品体验报告、产品结构分析、核心设计优缺点总结、可借鉴的设计方法、对己方产品的启示。

**文件结构**：
```
product-experience-report/
├── SKILL.md                              # 主文档：触发条件 + 工作流 + 参考索引
└── references/
    ├── template-five-layers.md            # 五要素每层分析维度 + checklist

```

## 触发场景

当用户需要系统性分析一款产品的用户体验时触发。典型场景包括：
- 深度拆解一款产品的设计思路
- 写产品体验报告 / 产品分析文章
- 面试前的产品分析准备
- 竞品体验对比
- 从一款产品中学习产品设计方法

## 输入要求

**必需信息**：
- 产品名称（至少 1 款）

**可选补充信息**：
- 分析侧重点（功能/设计/商业化/AI 能力等）
- 分析深度（快速扫描 / 深度体验）
- 输出形式（报告/PPT/面试答题）

如果信息不全，先补全再开始。

## 工作流（5 步，对应五要素）

### Step 1：战略层分析——为什么做

分析产品最底层的定位和目标：

**四个核心问题**：
1. **市场定位**：这款产品解决什么人的什么问题？处于什么赛道？
2. **目标用户**：核心用户是谁？有什么特征？
3. **商业目标**：产品靠什么赚钱？公司为什么做它？
4. **用户需求**：用户为什么用它？满足了什么需求？

**输出**：战略层分析结论 + 产品定位一句话总结

### Step 2：范围层分析——做什么功能

分析产品提供了哪些功能来实现战略：

1. **功能全景图**：按模块列出所有核心功能
2. **核心功能 vs 次要功能**：哪些是必须有的，哪些是锦上添花的
3. **AI 能力边界**（AI 产品适用）：AI 做了什么？没做什么？
4. **功能与战略的匹配度**：每个功能是否支撑战略目标？

**输出**：功能清单 + 功能模块分类 + 核心功能标注

### Step 3：结构层分析——功能怎么组织

分析功能如何组合成完整的产品：

1. **功能模块划分**：产品有几个大模块？每个模块负责什么？
2. **信息架构**：内容如何组织和分类
3. **用户流程图**：核心任务的完整流程
4. **AI 参与模式**（AI 产品适用）：
   - Copilot 辅助模式（AI 出建议，人工确认）
   - Agent 自主模式（AI 自动完成）
   - 插件工具模式（嵌入某个环节）

**输出**：模块结构图 + 核心用户流程图

### Step 4：框架层分析——页面怎么排版

分析每个页面的布局和交互设计：

1. **页面布局逻辑**：信息是怎么排布的？为什么这么排？
2. **导航设计**：用户如何在页面间跳转？
3. **交互设计**：关键操作的交互方式
4. **首屏设计分析**：首页各区域的设计目的和优缺点

**输出**：核心页面框架分析 + 交互优缺点

### Step 5：表现层分析——长什么样

分析产品的视觉设计：

1. **视觉风格**：整体风格定位（简约/活泼/专业/科技…）
2. **配色方案**：主色/辅助色/功能色，传递什么感觉
3. **品牌元素**：Logo、插画、动效等品牌表达
4. **视觉细节**：圆角、阴影、间距、字体等细节处理

**输出**：视觉设计总结 + 优缺点

### 最后：综合总结

- 产品整体体验评价
- 最值得学习的 3 个设计点
- 可以改进的 3 个地方
- 对我们的启示

## 五要素记忆口诀

> **战略层** = 为什么做（市场、用户、目标）
> **范围层** = 做什么功能（落地战略）
> **结构层** = 功能怎么拆分、页面分什么模块
> **框架层** = 页面怎么排版、交互怎么做
> **表现层** = 视觉、色彩、动效设计
