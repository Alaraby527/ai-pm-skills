# 用户访谈 Skill

## 简介

**干什么的**：从访谈提纲设计到洞察结论输出的完整用户访谈方法论工具包。覆盖半结构化访谈、5Why 追问法、出声思维、记录编码分析等核心方法。

**核心方法论**：6 步工作流（目标澄清→用户招募→提纲设计→访谈执行→记录编码→洞察输出）+ 5Why 追问法 + 三级编码分析法 + 访谈者实操技巧指南。

**你能得到什么**：访谈提纲、访谈记录整理与编码、用户洞察结论、痛点地图、需求优先级排序、行动建议。

## 触发场景

当用户需要通过访谈了解用户真实需求、痛点、行为时触发。典型场景包括：
- 设计用户访谈提纲 / 访谈问题
- 整理和分析访谈记录
- 准备用户访谈（招募、流程、技巧）
- 从访谈中提炼用户洞察
- 做需求调研前的访谈准备

## 输入要求

**必需信息**：
- 调研目标（为什么做访谈、要验证什么假设）
- 目标用户画像（访谈对象是谁）

**可选补充信息**：
- 访谈形式（线上/线下/电话）
- 访谈时长
- 样本量
- 是否已有部分用户信息或假设

如果信息不全，先通过对话补全再开始。

## 工作流（6 步）

### Step 1：明确调研目标与研究假设

- 明确访谈要回答的核心问题（3-5 个）
- 列出待验证的假设（每个假设可被证伪）
- 区分：探索性访谈（发现问题）vs 验证性访谈（验证假设）
- 确定输出形式和颗粒度

> 详见 `references/prompts.md` 中的目标澄清提示词

### Step 2：确定目标用户与招募标准

- 明确访谈对象的画像维度（行业、角色、使用频率、痛点类型等）
- 制定用户筛选标准（纳入标准 + 排除标准）
- 确定样本量（探索性 5-8 人可发现 80% 问题，验证性需更大样本）
- 设计招募话术和用户激励

### Step 3：设计访谈提纲

按标准结构设计：
1. **开场破冰**（5min）：自我介绍、说明目的、消除紧张、录音授权
2. **用户背景**（10min）：基本信息、使用产品/服务的经历
3. **使用场景深挖**（15-20min）：具体使用场景、任务流程、触发时机
4. **痛点与需求挖掘**（15-20min）：当前痛点、不满意的地方、理想中的样子
5. **方案/概念验证**（可选，10min）：展示原型/方案，收集反馈
6. **结尾**（5min）：补充问题、感谢、联系方式

> 完整模板见 `references/template-interview-guide.md`
> 各环节提示词见 `references/prompts.md`

### Step 4：执行访谈（实操指南）

**核心技巧**：
- **5Why 追问法**：连续追问 5 个"为什么"，挖到深层动机
- **区分说的和做的**：用户说的偏好 ≠ 实际行为，要问具体事例
- **避免引导性问题**：不用"你是不是觉得…"，改用"你当时怎么想的…"
- **沉默等待**：用户停顿后不要急于填补，给思考时间
- **出声思维法**：让用户边操作边说想法（适用于产品体验访谈）

**访谈者注意事项**：
- 多听少说，访谈者说话不超过 30%
- 不做产品推销，不辩解
- 随时记录关键引语（用户原话）
- 重要点当场确认，"我理解的是…对吗？"

### Step 5：访谈记录整理与编码

1. **逐字稿整理**：转写访谈录音，标记关键点
2. **开放式编码**：给每段有意义的对话贴标签（开放、自由命名）
3. **主轴编码**：将相似标签归类，提炼主题
4. **选择性编码**：提炼核心洞察，找到主题间的关联

编码方法参考：
- 按"行为 / 态度 / 痛点 / 需求 / 场景 / 动机"六类编码
- 标记频次和代表性引语
- 区分普遍问题 vs 个别问题

### Step 6：输出洞察结论

按 `references/template-interview-summary.md` 模板输出，包含：
- 调研背景与方法
- 用户画像总结
- 核心发现（3-5 个，每个配数据 + 引语）
- 用户痛点地图
- 需求优先级排序
- 行动建议
