# 用户访谈提示词合集

## 1. 调研规划阶段

### 1.1 调研目标澄清提示词

```
我想做【产品/功能】的用户访谈，我的目的是【简要描述你的目的，比如"了解用户怎么用这个功能"】。

请帮我：
1. 把这个目的拆解成 3-5 个具体的研究问题
2. 列出 3-5 个待验证的假设（每个假设必须是可证伪的）
3. 判断这是探索性访谈还是验证性访谈
4. 建议访谈人数和用户画像维度

【产品/功能简要描述：】
```

### 1.2 用户筛选标准生成提示词

```
我要做【调研主题】的用户访谈，目标用户是【描述你的目标用户】。

请帮我设计：
1. 用户纳入标准（至少 3 条）
2. 用户排除标准（至少 2 条）
3. 招募用户时的筛选问卷（5-8 个问题）
4. 建议的样本量和用户分层
```

## 2. 访谈提纲设计阶段

### 2.1 完整访谈提纲生成提示词

```
我要做【产品/服务】的用户访谈，调研目标是：
【1. ...
 2. ...
 3. ...】

目标用户画像是：
【描述用户画像】

请生成一份完整的访谈提纲，要求：
1. 包含 6 个标准环节：开场破冰、用户背景、使用场景、痛点挖掘、方案验证（可选）、结尾
2. 每个环节 5-10 个具体问题
3. 问题要开放、具体，避免引导性
4. 标注每个环节的建议时长
5. 对于关键问题，给出追问方向（至少 3 层追问）

产品/服务背景：
【补充背景信息】
```

### 2.2 5Why 追问设计提示词

```
我在访谈中想深挖用户的【某个行为/观点/痛点】：
【描述你想深挖的点】

请帮我设计一套 5Why 追问链：
1. 第一层问题（引出话题）
2. 第二层追问（为什么会这样）
3. 第三层追问（背后的原因）
4. 第四层追问（深层动机）
5. 第五层追问（根本原因）

每一层给 2-3 种不同问法，避免显得机械。
```

### 2.3 痛点挖掘问题设计提示词

```
我要调研【产品/功能】的用户痛点。目标用户是【用户画像】。

请帮我设计 10 个痛点挖掘问题，要求：
1. 从不同角度切入：使用前、使用中、使用后、对比替代方案
2. 问题要具体，避免"你觉得有什么痛点"这种空泛问法
3. 每个问题配 1-2 个追问方向
4. 标注每个问题主要挖掘什么类型的痛点

痛点类型参考：效率痛点、成本痛点、体验痛点、情感痛点、社交痛点
```

## 3. 记录整理与分析阶段

### 3.1 访谈记录结构化整理提示词

```
以下是一段用户访谈记录，请帮我结构化整理：

【粘贴访谈记录/逐字稿】

请输出：
1. 用户画像标签（10 个以内关键词）
2. 核心观点摘要（5-8 条，每条配原文引语）
3. 提到的痛点（按强度排序）
4. 提到的需求/建议
5. 有价值的"意外发现"

每条信息标注出现的大概位置（时间段/上下文）。
```

### 3.2 多场访谈编码分析提示词

```
以下是【数量】场用户访谈的要点摘要，请帮我做编码分析：

【U01：
 ...
 U02：
 ...
 ...】

请完成：
1. 开放式编码：提取所有有意义的观点，给每个观点一个编码标签
2. 主轴编码：将相似编码归类，提炼出 5-8 个主题
3. 统计每个主题被多少人提到（频次）
4. 每个主题配 1-2 条最有代表性的用户原话
5. 标注哪些是普遍问题（>50% 用户提到），哪些是个别问题

输出格式：表格 + 简要说明。
```

### 3.3 洞察提炼提示词

```
以下是用户访谈的编码分析结果：
【粘贴编码分析结果】

调研目标是：【调研目标】

请帮我提炼出 3-5 条核心洞察，要求：
1. 每条洞察不只是描述现象，要说出"为什么"和"意味着什么"
2. 每条洞察配数据支撑（多少人提到）和典型引语
3. 洞察要有启发性，能指导产品决策
4. 按照重要性排序

洞察公式：观察到的现象 + 背后的原因/动机 + 对产品的启示
```

## 4. 报告输出阶段

### 4.1 完整调研报告生成提示词

```
请基于以下访谈数据，生成一份完整的用户访谈报告：

【调研背景：】
【访谈对象：共 X 人，画像分布：...】
【核心发现/编码结果：】

报告结构要求：
1. 调研概况
2. 用户画像总结（2-3 个人物）
3. 核心发现（3-5 条，每条配数据+引语+分析）
4. 痛点地图（按优先级排列）
5. 需求清单与优先级
6. 行动建议（产品+运营+后续研究）

要求专业、务实，不说空话。
```
