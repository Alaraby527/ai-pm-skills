所有提示词均可直接复制使用，根据你的实际情况替换方括号中的内容。

## 一、简历优化类

### 提示词1：JD关键词提取

```
请帮我分析下面这份产品经理JD，提取出所有关键词，按"硬技能/业务场景/软素质"三类整理，并标注每个关键词的出现频次和重要性。

【JD内容】
[粘贴完整JD]

输出格式：
- 硬技能关键词（按重要性排序）：
- 业务场景关键词（按重要性排序）：
- 软素质关键词（按重要性排序）：
- 简历匹配建议（我应该重点突出哪些经历）：
```

### 提示词2：万能项目描述生成

```
请用"万能项目描述公式"帮我优化下面这个项目的描述。公式是：项目标题 + 背景 + 方案 + 核心亮点 + 数据结果 + 复盘认知。

要求：
1. 背景要讲清楚业务痛点和商业价值
2. 方案要体现思考深度，不是简单罗列功能
3. 亮点要突出创新点、难点突破或跨部门推动
4. 结果要量化，用对比数据凸显价值
5. 复盘要有产品认知升级，不是流水账
6. 整体语言要专业、精炼、有说服力
7. 针对目标岗位是AI产品经理，突出AI相关能力

【项目原始信息】
项目名称：[项目名]
你的角色：[产品负责人/核心产品等]
项目背景：[原始背景描述]
做了什么：[你做的具体事情]
结果数据：[有什么数据]
学到了什么：[你的思考]

【目标JD关键词】
[从JD提取的核心关键词]

请输出优化后的项目描述，分模块标注。
```

### 提示词3：自我评价生成

```
请帮我写一段产品经理简历的自我评价，2-3行。要求：

1. 采用两段式结构：
   - 第一段：经历概括（从业年限、深耕赛道、核心操盘模块）
   - 第二段：差异化亮点（大厂背书/从0到1/核心增长成果）

2. 每一句话都必须有项目经历可以佐证，不能是空话

3. 针对AI产品经理岗位，突出AI相关经验

【我的背景】
从业年限：[X年]
深耕赛道：[AI/大模型/智能客服等]
核心项目：[2-3个核心项目及数据成果]
差异化亮点：[大厂/从0到1/数据成果等]

输出精炼的自我评价，不要超过3行。
```

### 提示词4：简历整体诊断

```
请以资深HR和产品总监的双重视角，帮我诊断这份简历。

【我的简历】
[粘贴完整简历]

【目标岗位】
AI产品经理

请从以下维度诊断：
1. 结构清晰度（排版、分区、可读性）
2. JD匹配度（关键词覆盖、经验相关性）
3. 项目描述质量（是否有数据、有亮点、有逻辑）
4. 致命错误（错别字、空泛描述、格式问题等）
5. 整体评分（1-10分）和改进优先级

每条问题都给出具体的修改建议和前后对比示例。
```

## 二、面试准备类

### 提示词5：项目5维度素材梳理

```
请帮我把下面这个项目整理成面试用的5维度标准化素材。5个维度是：
1. 项目背景（行业现状、业务痛点、商业价值）
2. 项目目标（核心指标、指标逻辑、数据预估）
3. 产品方案（策略分层设计、需求对应关系、权衡取舍）
4. 难点 & 亮点（矛盾平衡、创新解法、跨资源推动）
5. 结果 & 复盘（数据对比、方法论沉淀、踩坑教训）

【项目原始信息】
[粘贴项目详细信息，越详细越好]

【要求】
- 每个维度整理成要点形式，面试时可直接调用
- 准备一个3分钟版本（自我介绍后讲项目用）
- 准备一个1分钟版本（被问到"简单说下这个项目"时用）
- 标注面试官可能深挖的10个问题和答题思路
```

### 提示词6：模拟面试（项目深挖）

```
请你扮演字节跳动的AI产品经理一面面试官，对我进行项目深挖面试。

【我的项目】
[粘贴你的5维度项目素材]

【面试规则】
1. 你先让我做3分钟项目介绍
2. 然后你连续追问8-10个项目深挖问题（由浅入深）
3. 我每回答一个，你给我打分（1-10分）和点评
4. 最后给我一个整体评估和改进建议

问题要覆盖：
- 立项原因和机会来源
- 目标设定的依据
- 方案设计的取舍逻辑
- 竞品对比
- 最大难点和解决方案
- 数据结果分析
- 复盘与反思
- 如果重来会怎么做

现在开始，请先让我做项目介绍。
```

### 提示词7：行业认知题答题框架生成

```
我下周要面试[公司名]的AI产品经理岗位，面试官可能会问行业认知类问题。请帮我准备以下3道题的答题框架：

【题目1】[比如：你怎么看AI助手这个赛道？]
【题目2】[比如：XX和YY的核心差异是什么？]
【题目3】[比如：如果你来做我们的产品，你会怎么做？]

【公司/产品背景】
[补充你了解的公司和产品信息]

要求：
- 每题都给出结构化的答题框架（有逻辑分层）
- 要有数据支撑或行业洞察（可以合理估算）
- 要有自己的观点和判断，不能只是复述常识
- 结尾要有升华，体现产品思维高度
```

### 提示词8：行为面试题 STAR+R 答案生成

```
请帮我把下面这个经历改写成行为面试的标准答案，使用STAR+R法则（Situation-Task-Action-Result-Reflection）。

【原始经历】
[描述你的一段经历，比如"推动了一个跨部门项目"]

【题目类型】
[比如：最大的挑战/从0到1的项目/和别人意见不一致/失败的项目]

要求：
1. Situation：背景清晰，有冲突和张力
2. Task：目标明确，有量化指标
3. Action：具体、有细节，占60%篇幅，突出"我"做了什么
4. Result：数据支撑，和目标对比
5. Reflection：产品认知升级，体现复盘能力（这是产品经理加分项）
6. 整体200-300字，面试时说1.5-2分钟
```

## 三、求职规划类

### 提示词9：三梯队岗位盘点规划

```
我现在是一名[当前岗位]，想跳槽到AI产品经理方向。请帮我制定一个三梯队的岗位盘点规划。

【我的背景】
- 从业年限：[X年]
- 当前行业：[行业]
- 核心能力：[数据分析/需求拆解/跨部门推动等]
- 项目经验：[2-3个核心项目]
- 目标城市：[城市]

【我的诉求】
- 最看重：[发展/薪资/权限/团队/稳定性，按优先级排]
- 期望薪资涨幅：[比如20-30%]
- 目标行业：[AI/大模型/智能客服等]

请帮我输出：
1. 我的五维度匹配分析（能力/经验/行业/业务级别/公司级别）
2. 适合我的三梯队公司/岗位清单（每类至少5个方向）
3. 三阶段投递策略建议（试水期/大规模投递期/冲刺期）
4. 我的核心优势和短板，以及补强建议
```

### 提示词10：求职漏斗复盘诊断

```
我已经投了[X]份简历，拿到了[Y]个面试，通过了[Z]个一面。请帮我做一个求职漏斗诊断。

【数据】
- 投递总数：[X]份
- 拿到面试：[Y]个 → 投递→面试转化率：[Y/X%]
- 一面通过：[Z]个 → 一面通过率：[Z/Y%]
- 二面通过：[W]个 → 二面通过率：[W/Z%]

【我的情况】
- 目标岗位：AI产品经理
- 工作年限：[X年]
- 投递方向：[描述一下都投了什么类型的公司]

【行业参考均值】
- 投递→面试：20%
- 一面通过率：35-40%
- 二面通过率：30-35%

请帮我分析：
1. 每一层转化率和行业均值对比如何？
2. 我最大的短板在哪一层？可能的原因是什么？
3. 针对每一层，给出具体的改进建议（至少3条）
4. 接下来2周的行动清单
```
