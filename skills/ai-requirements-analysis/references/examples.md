# 示例：AI 智能客服需求分析完整示例

> 本示例基于知识库中的方法论，演示 AI 需求分析的完整输出。为节省篇幅，节选核心部分。

---

## 一、需求背景

### 业务背景
某电商平台客服团队 50 人，日均接待咨询 5000+ 单。随着业务增长，客服人力成本逐年上升，用户等待时间长，满意度下降。希望引入 AI 智能客服降低人工成本、提升响应速度。

### 用户痛点
1. 客服响应慢，高峰期排队 10 分钟以上
2. 简单问题也要等人工，效率低
3. 客服水平参差不齐，回答质量不稳定
4. 夜间/节假日覆盖不足

### 需求目标
- 业务目标：70% 的常见问题由 AI 自动解答，人工成本降低 30%
- 用户目标：平均响应时间从 5 分钟降到 30 秒以内
- 成功衡量：AI 解答准确率 ≥ 85%，用户满意度 ≥ 4.2/5

---

## 二、用户与场景

### 目标用户
- 主要用户：电商平台 C 端消费者（全年龄段）
- 次要用户：客服运营人员（维护知识库、处理复杂问题）

### 典型使用场景

**场景一：订单查询**
- 触发：用户下完单后想知道什么时候发货
- 目标：快速获取订单状态
- 当前痛点：要排队等人工，等了半分钟只问一句话
- 期望：输入问题秒回，直接给答案

**场景二：售后咨询**
- 触发：用户收到商品有问题，想退换
- 目标：了解退换货流程，发起售后申请
- 当前痛点：流程复杂，客服要一步步引导
- 期望：AI 直接告诉怎么做，甚至帮我操作

---

## 三、AI 参与模式与边界

### 模式选择
**混合模式**：
- 简单问题 → Agent 自主模式（AI 全自动解答）
- 复杂问题 → 转人工
- 模糊问题 → Copilot 模式（AI 建议 + 用户选择）

### 分工边界

| 环节 | AI 做什么 | 人做什么 |
|------|-----------|---------|
| 意图识别 | 自动分类用户问题意图 | 处理识别错误的情况 |
| 简单咨询（订单查询/营业时间等） | 自动查询并回答 | 不参与 |
| FAQ 类问题 | 基于知识库自动回答 | 维护知识库内容 |
| 售后/退款等操作 | 引导用户操作，简单情况自动处理 | 复杂/特殊情况人工介入 |
| 投诉/情绪问题 | 安抚情绪后转人工 | 人工跟进处理 |
| 业务范围外问题 | 礼貌兜底回复 | 不参与 |

### 兜底方案
- AI 置信度 < 60% → 直接转人工
- 用户连续 2 次不满意 → 自动转人工
- 用户说"人工"或"转人工" → 立即转接
- 夜间 AI 全部接管，重要问题留到第二天人工回

---

## 四、核心功能方案

### 功能一：智能问答（FAQ + RAG）

**功能描述**：用户提问时，AI 基于知识库自动给出答案。

**技术方案**：
- 高频短问题 → FAQ 知识库（语义相似度匹配）
- 复杂政策类问题 → RAG 文档知识库（向量检索 + 大模型生成）
- 两类知识混合路由，优先 FAQ，FAQ 命中不了再走 RAG

**Prompt 设计要点**：
```
角色：你是 XX 平台的智能客服助手，专业、友好、有耐心。

任务：根据用户的问题和提供的参考资料，给出准确、简洁的回答。

规则：
1. 只回答参考资料中有的内容，不知道就说"抱歉，这个问题我暂时还不会，已为您转接人工客服"
2. 回答要简洁，重点突出
3. 涉及金额、时间等具体数字要准确
4. 态度友好，多用"您"
...
```

## 五、验收指标

### 效果指标
| 指标 | 目标值 | 测量方式 |
|------|--------|---------|
| 问答准确率 | ≥ 85% | 人工抽样评测（每日 100 条） |
| 意图识别准确率 | ≥ 90% | 标注评测集 |
| 用户满意度 | ≥ 4.2/5 | 会话结束后评价 |

### 效率指标
| 指标 | 目标值 | 测量方式 |
|------|--------|---------|
| 平均响应时间 | ≤ 30 秒 | 系统日志 |
| 问题解决率 | ≥ 70% | 无需转人工的会话比例 |
| 人工接待量下降 | ≥ 30% | 对比上线前 |

### 业务指标
| 指标 | 目标值 | 测量方式 |
|------|--------|---------|
| 客服人力成本降低 | ≥ 30% | 财务数据 |
| 客服满意度提升 | 从 4.0 到 4.2 | 满意度调研 |
| 夜间服务覆盖率 | 从 0% 到 100% | 运营数据 |

## 六、MVP 规划

### MVP 功能范围
**必须有**：
1. FAQ 知识库问答（Top 50 问题）
2. 订单查询（对接订单系统）
3. 转人工功能
4. 基础知识库管理后台

**可以没有**：
- RAG 长文档问答
- 多轮复杂对话
- 主动营销
- 多语言

### MVP 验证目标
- 验证：Top 50 问题 AI 解答准确率 ≥ 80%
- 验证：用户接受度（满意度 ≥ 4.0）
- 周期：2 周开发 + 2 周灰度测试

### 迭代路线
- V1.0 (MVP)：FAQ + 订单查询 → 覆盖 30% 问题
- V1.1：RAG 知识库 → 覆盖 50% 问题
- V1.2：售后流程引导 → 覆盖 60% 问题
- V2.0：Agent 自主操作 + 多轮对话 → 覆盖 70%+
