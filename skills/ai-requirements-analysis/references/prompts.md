# AI 需求分析提示词合集

## 1. 需求澄清阶段

### 1.1 AI 需求四问澄清提示词

```
我有一个 AI 产品想法：【描述你的想法，越详细越好】

请用 AI 需求四问法帮我澄清这个需求：

1. 用户是谁？
   - 目标用户画像（角色、行业、痛点）
   - 有几类用户？谁是核心用户？
   
2. 什么场景下触发？
   - 典型使用场景
   - 触发条件
   - 使用频率
   
3. 当前怎么解决的？
   - 现有方案
   - 现有方案的痛点
   - 替代方案对比

4. AI 能解决吗？
   - AI 在这个需求里的角色
   - 哪些 AI 能做，哪些不能
   - 做不到时怎么兜底

请尽量具体，不要空泛。如果信息不足，请指出我需要补充什么。

【业务背景补充：】
```

### 1.2 需求拆解提示词

```
这是一个 AI 产品需求：【需求描述】

请帮我把这个需求拆解成：
1. 用户痛点列表（至少 5 个，按严重程度排序）
2. 功能点列表（对应每个痛点的解决方案）
3. 优先级排序（P0/P1/P2）及判断理由
4. MVP 版本应该做什么、不做什么
5. 成功标准（怎么判断这个需求做对了）
```

## 2. 可行性评估阶段

### 2.1 技术可行性分析提示词

```
我想做一个 AI 功能：【功能描述】

请帮我做技术可行性分析：

1. 模型能力评估
   - 当前主流大模型（GPT-4、Claude、文心、通义等）能做到吗？
   - 准确率/效果大概能到什么水平？
   - 有哪些已知的能力边界？

2. 技术路线选择
   - 纯 Prompt 工程能实现吗？
   - 是否需要 RAG？为什么？
   - 是否需要微调？为什么？
   - 是否需要自研模型？为什么？
   - 推荐技术路线及理由

3. 数据需求
   - 需要什么数据？
   - 数据从哪里来？
   - 数据获取难度如何？

4. 实现复杂度
   - 预估开发周期
   - 需要什么角色的人
   - 主要技术难点

5. 风险点
   - 最大的技术风险是什么？
   - 怎么规避或验证？

我的背景：【描述你的技术资源/团队情况】
```

### 2.2 成本可行性分析提示词

```
【功能描述】这个 AI 功能，请帮我估算成本：

1. 开发成本
   - 需要什么技术角色
   - 预估人天
   - 总成本估算

2. 运行成本
   - 单次调用成本（Token 估算）
   - 假设日活 X，月度成本估算
   - 成本优化空间（缓存、模型分层等）

3. 投入产出比
   - 对比纯人工方案的成本
   - 多久能回本
   - ROI 估算

4. 不同方案的成本对比
   - 方案 A（纯 Prompt）
   - 方案 B（RAG）
   - 方案 C（微调）
```

### 2.3 合规风险评估提示词

```
【AI 功能/产品描述】

请帮我评估这个 AI 产品的合规风险：

1. 数据隐私风险
   - 涉及哪些用户数据？
   - 是否需要用户授权？
   - 数据能不能用于模型训练？
   - 跨境数据传输问题？

2. 内容安全风险
   - 可能生成哪些有害内容？
   - 风险等级（高/中/低）
   - 如何防范？

3. 行业监管
   - 【行业】有哪些特殊的监管要求？
   - AI 相关的政策法规影响？

4. 伦理风险
   - 可能存在哪些偏见/歧视问题？
   - 透明度问题（用户知不知道是 AI？）
   - 责任归属问题

5. 风险应对建议
```

## 3. 方案设计阶段

### 3.1 AI 参与模式选择提示词

```
【产品/功能描述】

请帮我判断这个 AI 功能应该用哪种参与模式：

模式选项：
1. Copilot 辅助模式 —— AI 出草稿/建议，人工最终确认
2. Agent 自主模式 —— AI 自动完成全流程，自主决策
3. 插件工具模式 —— AI 仅作为单一环节嵌入业务流程

请分析：
1. 每种模式的适用情况
2. 推荐哪种模式？为什么？
3. 如果选 Copilot，AI 和人的分工边界是什么？
4. 如果选 Agent，异常兜底方案是什么？
5. 如果选插件，嵌入哪个环节？上下游是什么？

判断维度建议：错误成本、用户预期、自动化程度、技术成熟度
```

### 3.2 MVP 方案设计提示词

```
【产品/需求描述】

我想做 MVP 版本，请帮我规划：

1. MVP 的核心目标是什么？（验证什么假设？）
2. MVP 必须有的 3 个核心功能
3. MVP 可以没有的功能（即使很想要也先砍掉）
4. MVP 的最简实现路径
   - 能不能用现成工具（Coze/Dify 等）先搭？
   - 哪些可以用人工替代？
   - 最快多久能上线？

5. MVP 的成功标准
   - 看什么指标？
   - 达到什么数值算成功？
   - 验证周期多长？

6. MVP 之后的迭代路线图
   - V1.0 → V1.1 → V2.0 各加什么
```

### 3.3 Prompt 方案设计提示词

```
我需要为【功能场景】设计 Prompt。

功能描述：
【详细描述这个 AI 功能做什么、输入什么、输出什么】

请帮我设计：
1. Prompt 整体框架
   - 角色设定
   - 任务指令
   - 输入格式
   - 输出格式
   - 约束条件

2. 第一版完整 Prompt

3. 测试这个 Prompt 可能遇到的问题
   - 哪些情况可能输出不符合预期？
   - 如何优化？

4. 如果效果不好，优化方向有哪些？
```

## 4. 文档输出阶段

### 4.1 PRD 生成提示词

```
请帮我把以下需求整理成一份完整的 AI 产品 PRD：

【需求背景：】
【目标用户：】
【核心功能：】
【技术方案：】
【验收标准：】

按照标准 AI PRD 模板输出，包含：
1. 需求背景与目标
2. 用户与场景
3. AI 参与模式与边界
4. 功能方案（核心功能详细说明）
5. 技术可行性分析
6. 验收指标（效果/效率/业务三维）
7. MVP 版本规划
8. 风险与应对
9. 合规与安全

要求专业、具体、可执行。
```
