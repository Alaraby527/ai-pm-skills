# 调研问卷设计 Skill

## 简介

**干什么的**：用户研究问卷设计与数据分析方法论工具包。从调研目标到问卷题目设计、信效度保障、交叉分析框架，输出完整问卷与分析方案。

**核心方法论**：6 步问卷设计工作流 + 题目设计原则与题型选择指南 + 信效度保障方法 + 交叉分析框架（描述统计→交叉分析→相关性分析→洞察提炼）。

**你能得到什么**：完整调研问卷、问卷设计优化建议、问卷数据分析报告（含交叉分析）、用户分层结果、调研结论与建议。

**文件结构**：
```

├── SKILL.md                              # 主文档：触发条件 + 工作流 + 参考索引
└── references/
```

## 触发场景

当用户需要设计问卷或分析问卷数据时触发。典型场景包括：
- 设计用户满意度/产品使用调研问卷
- 设计市场调研问卷
- 问卷题目设计与优化
- 问卷数据分析与交叉分析
- 写问卷调研报告

## 输入要求

**必需信息**：
- 调研目标（为什么做问卷）
- 待验证的假设（或要回答的问题）
- 目标用户群体

**可选补充信息**：
- 样本量预估
- 发放渠道
- 已有假设/前期调研发现

如果信息不全，先通过对话补全。

## 工作流（6 步）

### Step 1：明确调研目标与假设

- 明确问卷要回答的核心问题（3-5 个）
- 列出研究假设（每个假设可被数据验证或证伪）
- 确定需要收集的数据类型（行为数据/态度数据/人口统计）
- 确定输出物形式和颗粒度

### Step 2：设计问卷整体结构

标准结构：
1. **甄别题**（2-3题）：筛选目标用户，不符合的直接结束
2. **人口统计题**（3-5题）：性别、年龄、职业、城市等（按需）
3. **核心行为题**（5-10题）：使用频率、使用场景、功能使用等
4. **态度题**（5-10题）：满意度、NPS、态度量表等
5. **开放题**（1-2题）：收集未覆盖的意见和建议

> 问卷总时长控制在 5-10 分钟（约 20-30 题），太长会降低完成率

### Step 3：编写具体题目

**题目设计原则**：
- 避免双重含义（一个问题只问一件事）
- 避免引导性/倾向性措辞
- 选项互斥且穷尽
- 量表一致性（统一用 5 点或 7 点 Likert）
- 敏感问题后置
- 按逻辑顺序排列（从易到难、从行为到态度）

**常见题型选择指南**：
| 要测什么 | 推荐题型 |
|---------|---------|
| 使用频率/时长 | 单选题（选项分段） |
| 满意度 | Likert 量表（5点/7点） |
| 忠诚度 | NPS（0-10分） |
| 功能重要性 | 重要-满意度矩阵题 |
| 多选问题 | 多选题（注意选项不重叠） |
| 开放意见 | 开放题（1-2道足够） |

> 详见 `references/template-questionnaire.md`

### Step 4：预测试与优化

- 找 5-10 个目标用户做预测试
- 检查：题目是否理解一致？有没有歧义？有没有答不出来的？
- 检查完成时长（超时要删题）
- 修改后定稿

### Step 5：发放与数据回收

- 选择发放渠道（问卷平台 / 产品内弹窗 / 社群 / 邮件）
- 设置质量控制（答题时间过短的标记为无效）
- 监控回收进度和样本构成
- 目标样本量计算参考：
  - 描述性研究：至少 100 份
  - 对比分析：每组至少 30 份
  - 因子分析/聚类分析：至少 200 份

### Step 6：数据分析与报告输出

**分析框架**：
1. **描述统计**：各题的分布、均值、占比
2. **交叉分析**：
   - 用户分层 × 满意度（不同人群满意度差异）
   - 使用行为 × 态度（高频用户和低频用户的态度差异）
   - 人口统计 × 行为（不同年龄/性别的使用差异）
3. **相关性分析**：哪些因素和满意度/NPS 最相关
4. **洞察提炼**：从数据到结论到建议

> 详见 `references/template-survey-analysis.md` 和 `references/prompts.md`
