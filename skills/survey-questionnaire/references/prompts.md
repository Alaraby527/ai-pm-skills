# 调研问卷提示词合集

## 1. 问卷设计阶段

### 1.1 问卷结构规划提示词

```
我要做一个关于【调研主题】的问卷调研。

调研目标：
1. 【目标 1】
2. 【目标 2】
3. 【目标 3】

目标用户：【描述目标用户】

请帮我规划这份问卷：
1. 问卷应该包含哪几个部分？每部分的目的是什么？
2. 预估总共多少道题？完成时间多长？
3. 哪些信息是必须收集的？哪些是可选的？
4. 建议用什么题型？为什么？
5. 为了达到调研目标，最关键的 3-5 道题是什么？

输出格式：问卷结构大纲，包含每部分的题目类型、题量、目的。
```

### 1.2 完整问卷生成提示词

```
请帮我设计一份完整的用户调研问卷。

【调研主题】：
【调研目标】：
  1. ...
  2. ...
  3. ...
【目标用户】：
【预计完成时长】：5-8 分钟
【问卷平台】：（可选，问卷星/腾讯问卷/金数据等）

要求：
1. 包含完整的 5 个部分：甄别题、人口统计题、核心行为题、态度题、开放题
2. 每道题给出完整的选项
3. 标明哪些题是必答题
4. 总题量控制在 20-25 道
5. 题目设计符合问卷设计原则（无引导性、选项互斥穷尽、量表一致等）
6. 在关键题目旁边加一句【设计说明】，解释为什么这么设计

请直接输出问卷，格式清晰。
```

### 1.3 量表题设计提示词

```
我想在问卷中测量【测量对象，比如"用户满意度"】，请帮我设计量表题。

要求：
1. 给出推荐的量表类型（Likert 5点/7点、语义差异、NPS 等）
2. 设计 5-8 个测量题项
3. 每个题项要具体、可操作，避免抽象
4. 题项从不同维度覆盖测量对象
5. 说明计分方法和结果解释

【测量对象详细描述：】
```

### 1.4 问卷优化/审校提示词

```
这是我设计的一份问卷，请帮我审校和优化：

【粘贴问卷】

请从以下角度审查：
1. 题目有没有歧义？有没有双重含义？
2. 有没有引导性或倾向性措辞？
3. 选项是否互斥且穷尽？
4. 量表使用是否一致？
5. 题目顺序是否合理？
6. 题量是否合适？哪些题可以删？
7. 有没有遗漏重要的维度？
8. 开放题设计是否合理？

输出：
- 问题清单（每条标注：问题描述 + 修改建议）
- 优化后的完整问卷
```

## 2. 样本量与投放阶段

### 2.1 样本量计算提示词

```
我要做一个关于【调研主题】的问卷调研，请帮我估算需要多少样本量。

调研目的：【描述目的，如满意度调研/用户分层/对比分析】
目标用户总数：【总体规模，不知道可以写"未知"】
置信水平：95%（默认）
可接受误差范围：±5%（默认）

请帮我算：
1. 理论最小样本量
2. 考虑无效问卷后的建议样本量
3. 如果要做子群体对比分析（按年龄/城市/使用频率等），每组需要多少样本
4. 样本量不够时的替代方案

参考公式说明即可，不需要精确到个位。
```

## 3. 数据分析阶段

### 3.1 数据分析思路提示词

```
我有一份问卷调查的数据，调研主题是【主题】，调研目标是【目标】。

问卷的核心维度有：
1. 【维度 1】
2. 【维度 2】
3. 【维度 3】
...
收集到的样本量是 XX 份，用户画像分布大概是【描述分布】。

请帮我设计一个数据分析框架：
1. 描述统计层面，应该看哪些指标？
2. 交叉分析层面，应该做哪些交叉组合？为什么？
3. 哪些发现是"值得深挖"的信号？
4. 最终报告应该有哪些核心结论模块？

重点：结合调研目标，不要为了分析而分析。
```

### 3.2 交叉分析提示词

```
以下是问卷数据的统计结果，请帮我做交叉分析：

【粘贴数据摘要，如：
- 整体满意度：3.8/5
- 高频用户占比 40%，满意度 4.2；低频用户占比 60%，满意度 3.5
- 一线城市用户 NPS 20，三四线 NPS -10
- 功能 A 使用率 80%，满意度 3.2；功能 B 使用率 30%，满意度 4.5
...】

请输出：
1. 最值得关注的 3 个交叉发现
2. 每个发现的可能原因分析
3. 对产品/运营的启示
4. 如果有疑问，指出还需要补充什么数据来验证

重点：不要只是描述差异，要分析"为什么有这个差异"和"意味着什么"。
```

### 3.3 重要性-满意度分析提示词

```
以下是各功能/维度的重要性和满意度数据：

【功能/维度】  【重要性(1-5)】  【满意度(1-5)】
功能 A          4.5              3.2
功能 B          4.2              4.3
功能 C          3.8              2.9
功能 D          3.5              4.0
功能 E          2.8              3.5
...

请帮我做重要性-满意度矩阵分析：
1. 将各功能放入四个象限（优势区/改进区/机会区/维持区）
2. 每个象限的策略建议
3. 优先级排序（先做什么、后做什么）
4. 输出一个可视化的矩阵描述（文字版即可）
```

### 3.4 开放题分析提示词

```
以下是用户开放题的回答汇总（共 XX 条）：
【粘贴开放题答案，或提炼后的要点】

请帮我做定性分析：
1. 归纳出主要的主题/类别（每个类别有多少条反馈）
2. 每个类别下面最有代表性的 2-3 条原话
3. 正负面反馈的比例
4. 有什么"意外"的反馈（问卷没覆盖到的）
5. 对产品改进的启示

要求：分类清晰，每个类别配数据和原话。
```

## 4. 报告输出阶段

### 4.1 完整调研报告生成提示词

```
请基于以下问卷数据，生成一份完整的问卷调研报告：

【调研背景】：
【样本概况】：
【核心数据摘要】：
  - 使用行为：
  - 满意度：
  - NPS：
  - 交叉分析发现：
  - 开放题反馈：

报告结构要求：
1. 调研概况
2. 描述统计结果（行为 + 态度）
3. 交叉分析（至少 3 个维度的交叉）
4. 核心发现（3-5 条，每条配数据+分析+启示）
5. 开放题分析
6. 行动建议（产品+运营+后续研究）

要求：
- 专业、务实，有数据支撑
- 结论明确，不说空话
- 建议具体、可执行
- 按照重要性排序
```
