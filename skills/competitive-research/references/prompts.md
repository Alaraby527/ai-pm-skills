# 竞品调研提示词合集

## 1. 调研规划阶段

### 1.1 竞品筛选提示词

```
我要做【产品/赛道名称】的竞品调研，调研目标是【具体目标】。

请帮我：
1. 列出这个赛道的主要玩家，分为直接竞品、间接竞品、潜在竞品三类
2. 每类推荐 3-5 个最值得研究的产品，并说明入选理由
3. 根据调研目标，建议重点深入研究哪 2-3 个核心竞品
```

### 1.2 调研提纲生成提示词

```
我要调研【产品名】，它的定位是【定位描述】，对标【我方产品/赛道】。

请生成一份完整的调研提纲，包含：
1. 基础信息维度（产品定位、用户、定价等）
2. 功能模块拆分清单
3. AI 能力测试项（至少 10 个具体测试用例）
4. 技术壁垒分析维度
5. 需要重点关注的差异化点

要求具体、可执行，不要空泛的条目。
```

## 2. 2C 产品实测阶段

### 2.1 核心功能体验测试提示词

```
我正在测试【产品名】的【功能模块】。请帮我设计 5-8 个测试用例，要求：
1. 覆盖核心场景、边缘场景、异常场景
2. 每个用例包含：测试目标、输入内容、预期输出、评估标准
3. 能有效测试出产品的真实能力边界

【产品功能简要描述：】
```

### 2.2 横向对比测试提示词

```
我要横向对比【产品 A】和【产品 B】在【具体能力】上的差异。

请设计一个公平的对比测试方案：
1. 设计 3-5 个标准测试 prompt（同样的输入给两个产品）
2. 给出评分维度和评分标准
3. 说明如何判断哪个产品更优

【能力描述：】
```

### 2.3 模型能力边界探测提示词

```
我想测试这款 AI 产品的能力边界。请帮我设计 10 个递进式的测试用例，从简单到困难，用于探测：
1. 基础理解能力
2. 长文本处理能力
3. 逻辑推理能力
4. 事实准确性
5. 创意生成能力
6. 格式遵循能力
7. 多轮上下文能力
8. 中文理解与生成
9. 代码/数学等专业能力
10. 安全与合规能力

每个用例给一个具体的测试 prompt。
```

## 3. 2B 产品调研阶段

### 3.1 2B 产品信息梳理提示词

```
请帮我梳理【产品名】这款 2B AI 产品的完整信息，输出结构化的分析，包含：
1. 公司背景与产品定位
2. 目标客户与典型行业
3. 核心功能模块（按模块分类）
4. 价格体系（不同套餐的差异）
5. 客户案例（至少 3 个，说明行业、场景、效果）
6. 集成方式与开放能力
7. 合作伙伴生态
8. 核心优势与可能的短板

基于公开信息合理推断，不确定的地方标注「推断」。
```

### 3.2 技术栈推断提示词

```
基于以下关于【产品名】的公开信息，请帮我推断其可能的技术栈和架构：
【粘贴公开信息：官网描述、白皮书、技术博客、招聘信息等】

请分析：
1. 基础模型选型（自研/开源/API调用）
2. RAG 实现方案推测
3. 工程架构特点
4. 可能的技术壁垒
5. 团队配置推断
```

## 4. 分析输出阶段

### 4.1 竞品矩阵分析提示词

```
我已经收集了【竞品数量】个产品的信息，请帮我整理成一份竞品对比分析。

【产品 1 信息：】
【产品 2 信息：】
【产品 3 信息：】
...

请输出：
1. 对比矩阵表格（维度：定位、目标用户、核心功能、AI能力、定价、优缺点）
2. 每家产品的核心竞争力分析
3. 各家的差异化定位总结
4. 如果我们要做，机会点在哪里
```

### 4.2 技术壁垒分析提示词

```
基于以下竞品调研信息，请帮我深度分析各家的技术壁垒：

【调研信息摘要：】

请回答：
1. 哪些能力是真的有技术门槛的？门槛在哪里？
2. 哪些是"看起来难但其实可以快速跟进"的？
3. 数据飞轮效应是否存在？体现在哪里？
4. 如果我们要做，最难突破的壁垒是什么？
5. 有什么"取巧"的路径可以绕开壁垒？
```

### 4.3 破局机会分析提示词

```
这是【赛道名称】的竞品分析结论：
【粘贴分析结论】

我们的优势/资源是：【我方资源描述】

请帮我分析：
1. 这个赛道还有哪些差异化机会？
2. 最适合我们的切入点是什么？为什么？
3. 应该避开哪些"看起来美但其实是坑"的方向？
4. MVP 版本应该做什么、不做什么？
5. 3 个月内可以验证什么假设？
```
