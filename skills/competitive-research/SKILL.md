# 竞品调研 Skill

## 简介

**干什么的**：系统性调研竞品、摸清赛道竞争格局的方法论工具包。从调研目标拆解到竞品矩阵输出，覆盖 2C 产品实测与 2B 产品深研两条路径。

**核心方法论**：6 步调研工作流 + 三层竞品分类法 + 2C/2B 双路径深度测评 + 技术壁垒与差异化分析框架。

**你能得到什么**：结构化竞品调研报告、竞品对比矩阵、技术壁垒评估、破局机会与优先级建议。

**文件结构**：
```

├── SKILL.md                          # 主文档：触发条件 + 工作流 + 参考索引
└── references/

```

## 触发场景

当用户需要系统性了解某个产品、品类或赛道的竞争格局时触发。典型场景包括：
- 明确说「做竞品调研」「竞品分析」
- 要对比两个或多个产品的差异
- 想了解某款产品的能力水平、技术壁垒
- 进入新赛道前的竞争格局扫描
- 面试前的竞品分析准备

## 输入要求

**必需信息**：
- 调研目标（为什么做、决策用）
- 至少 1 个核心竞品名称或品类方向

**可选补充信息**：
- 己方产品/方案定位
- 关注的重点维度（功能 / 技术 / 定价 / 用户体验 等）
- 输出形式偏好（报告 / 表格 / 汇报用）

如果信息不全，先通过对话补全再开始。

## 工作流（6 步）

### Step 1：明确调研目标与范围

- 确认调研目的：立项参考 / 功能对标 / 面试准备 / 战略分析
- 划定范围：直接竞品 / 间接竞品 / 潜在竞品 / 全赛道扫描
- 确定输出物形式和颗粒度

### Step 2：锁定竞品清单

按三层分类法筛选竞品：
1. **直接竞品**：同赛道、同目标用户、同类产品（如飞书 vs 钉钉）
2. **间接竞品**：解决同一问题但产品形态不同（如飞书 vs 企业微信）
3. **潜在竞品/标杆**：不同赛道但方法论可借鉴（如 Notion 的 AI 功能设计）

输出：竞品初选清单（3-8 个，含入选理由）

### Step 3：基础信息收集与功能对比

对每个竞品收集：
- 产品定位与目标用户
- 核心功能清单（按模块分类）
- AI 能力覆盖范围
- 定价策略与商业模式
- 用户规模与市场份额（可获取时）
- 优缺点初步判断

输出：竞品对比矩阵（详见 `references/template-competitor-matrix.md`）

### Step 4：深度测评（2C / 2B 两条路径）

**路径 A：2C 产品实测（5 步法）**
1. 注册走通完整新手流程
2. 核心功能全流程体验
3. 设计边界 case 测试模型能力
4. 同一问题/任务横向对比输出质量
5. 按能力等级评分（S/A/B/C）

**路径 B：2B 产品调研**
1. 官网 / 白皮书 / 客户案例梳理
2. 价格体系与套餐对比
3. 集成方式与开放能力分析
4. 行业解决方案覆盖度
5. 技术栈与合作伙伴推断

> 详见 `references/prompts.md` 中的测评提示词模板

### Step 5：技术壁垒与差异化分析

回答三个核心问题：
1. **技术壁垒**：哪些能力是真的有门槛（数据飞轮 / 算法专利 / 基础设施），哪些是可以快速跟进的？
2. **差异化定位**：各竞品的核心差异在哪里（用户群 / 价格带 / 功能侧重 / 商业模式）？
3. **破局机会**：如果我们做，切入点在哪里？避开什么？

### Step 6：输出调研报告

按 `references/template-research-report.md` 模板输出结构化报告，包含：
- 调研背景与目标
- 竞品全景图
- 核心竞品深度分析
- 横向对比矩阵
- 技术壁垒评估
- 结论与建议（含破局机会、优先级排序）
